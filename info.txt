1. PROJECT OVERVIEW
------------------
Placement Tracker is an AI-powered preparation platform designed to help students 
track their DSA progress, analyze resumes using Machine Learning, and simulate 
mock interviews. It features a microservice architecture for scalability.

2. TECH STACK & ARCHITECTURE (VIVA EXPLANATION)
-----------------------------------------------

A. FRONTEND: React (Vite, Tailwind CSS, Framer Motion)
   - Why React? Component-based architecture allows building a modular and 
     reusable UI. Virtual DOM ensures high performance for real-time dashboards.
   - Why Vite? Extremely fast build tool and development server compared to CRA.
   - Why Tailwind? Utility-first CSS for rapid, responsive, and modern UI design.
   - Why Framer Motion? To provide smooth, professional animations.

B. BACKEND: Spring Boot (Java, Spring Security, JPA)
   - Why Spring Boot? Robust, enterprise-grade framework. Handles complex 
     business logic and multi-module integration.
   - Why Spring Security + JWT? Stateless authentication is perfect for 
     microservices. Tokens ensure secure communication between services.

C. ML SERVICE: Python (Django REST Framework, PyPDF2)
   - Why Python? The industry standard for AI/ML. 
   - Why Django? Provides a secure and scalable way to expose ML models as 
     REST APIs quickly.

D. DATABASE & CACHE: PostgreSQL & Redis
   - Why PostgreSQL? Reliable relational database for structured data.
   - Why Redis? High-performance caching to speed up leaderboard calculations.

E. DEVOPS: Docker & Docker Compose
   - Why Docker? Ensures the app runs identically everywhere by containerizing 
     the frontend, backend, and ML services.

3. PROJECT FLOW & ARCHITECTURE
-----------------------------

A. AUTHENTICATION FLOW (JWT)
   1. User submits Login/Signup credentials via React.
   2. Spring Boot validates credentials and generates a JWT (JSON Web Token).
   3. Token is sent to the Frontend and stored in 'localStorage'.
   4. Axios Interceptor automatically attaches this token to the 'Authorization' 
      header for every subsequent API request.
   5. Spring Boot 'JwtAuthFilter' validates the token before allowing access 
      to protected resources.

B. RESUME ANALYZER FLOW
   1. User uploads a PDF resume in the Frontend.
   2. React sends the file as 'multipart/form-data' to the Spring Boot backend.
   3. Spring Boot forwards the file to the Django ML Service (http://ml-service:8000/analyze/).
   4. Django parses the PDF using PyPDF2, extracts skills, and calculates an 
      ATS score based on industry keywords.
   5. Results are sent back through Spring Boot to the React UI for display.

C. DASHBOARD ANALYTICS FLOW
   1. Frontend requests analytics data from '/api/analytics'.
   2. Spring Boot queries PostgreSQL for user's solved problems and logs.
   3. It calculates distribution stats (Easy/Medium/Hard) and readiness scores.
   4. React receives the JSON and renders it using 'Recharts' within 
      ResponsiveContainers to ensure correct scaling.

D. DOCKER ARCHITECTURE
   - The project uses a bridge network where services communicate by name:
     - 'frontend' (Port 5173)
     - 'backend' (Port 8081)
     - 'ml-service' (Port 8000)
     - 'postgres' (Port 5432)
     - 'redis' (Port 6379)

4. HOW TO RUN
------------
1. Install Docker Desktop.
2. Run `docker-compose up --build` in the root folder.
3. Access at http://localhost:5173

5. THE PRS FORMULA
-----------------
PRS = (Accuracy * 30%) + (Difficulty * 25%) + (Consistency * 20%) + 
      (Time Efficiency * 15%) + (Core Coverage * 10%)
